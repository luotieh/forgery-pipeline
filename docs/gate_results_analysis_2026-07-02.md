## 2. 逐闸门分析

### Gate 1 — WEAK：分桶弱、回归强（分类器不匹配，非信号问题）
**现象**：3 类平衡准确率 0.472（落在 WEAK 区 [0.45,0.55)），但强度回归的 Spearman ρ 高达 **0.89**；多 σ(0.472) > 单 σ(0.340)。

**根因（②+①）**：mock 的 img2img 是**线性混合** `fake=(1−s)·orig+s·synth`，残差剖面随 s **单调**，故最小二乘回归轻松拿到 ρ=0.89；但同一特征上用 **NearestCentroid**（欧氏到类心）对一个近 1 维的**序数**信号做 3 分类效果差 → ba 低。**即 ba 与 ρ 的落差来自分类器选择，不是信号缺失**。同时 ρ=0.89 本身是 **mock 假象**：真实 SDEdit 在低强度 + VAE 往返后会变噪，真实 ρ 预计明显下降（清单也说「细粒度回归未必准，粗桶能分即过」）。

**优化建议**：
- **（②现在可做）** gate1 增加「回归→分桶」路径：把 `linear_fit_predict` 的预测强度按阈值分桶得到 ba，与直接分类取优；单调信号下 ba 会向 ρ 对齐、大概率跨过 0.55。
- **（②）** 或换有序分类器（1 维 logistic / 分位阈值），并把特征从 `[mean,std]` 扩到**残差分位数**（p10/p50/p90）以增判别力。
- **（③）** 结论以**真实 SD2 + 真实 SDEdit 数据**为准；届时重点看**粗桶 ba** 而非 ρ 的漂亮值。

### Gate 2 — WEAK：全局特征分不开算子（特征粒度问题）
**现象**：同模型 0.327 ≈ 跨模型 0.328，双双高于随机 0.20、低于 PASS 0.50；**same≈cross 说明不是 confound**（confound 应是 same 高、cross 低）。

**根因（②）**：`profile` 是**整图** residual 的 mean/std。而 5 个算子的差异主要在 **mask 的空间几何**（`box/border/invert`）与局部内容，**全局统计量天然分不开** inpaint/outpaint/replace/background；只有 img2img（全图）与局部算子在全局量上略有别。

**优化建议**：
- **（②现在可做，收益最大）** 加**空间/区域特征**：残差图高值区的几何（重心、扩散半径、**边界带 vs 中心区**残差比）、连通域形状、mask 支撑描述子——直接对应算子的掩码几何。
- **（②）** 先做 **img2img(全图) vs 局部** 二分层，再在局部内细分 4 算子。
- **（③）** mock 算子是粗色块，**低估**真实可分性；真实扩散算子签名更强，但全局特征的局限对真假数据都成立——所以先补空间特征，再上真实数据。

### Gate 3 — PASS：多σ增量真、跨生成器"无掉点"是假象
**现象**：多 σ 增量 **+0.132**（>0）；跨生成器 seen 0.371 → heldout 0.360，**drop 仅 0.011**。

**根因**：
- **（②真结论）** 多 σ 相对单 σ 有正增量——即便在弱分类器下也成立，说明「多尺度」这个词在管线上站得住（待真实信号复核）。
- **（①mock 假象）** 几乎无跨生成器掉点，是因为 mock 的留出生成器（`sdxl-img2img`/`kandinsky-inpaint`）产出的合成图与 seen 生成器**模式雷同**；真实不同模型族签名不同，**预计会有掉点**。
- **（护栏）** 基础算子准确率仅 0.36（近随机 0.20），**在如此低的绝对准确率下，drop 数值不可解读**。

**优化建议**：
- **（②）** 先按 Gate 2 的建议**修好特征**，把算子准确率抬到显著高于随机，再谈 drop；否则 drop≈0 无意义。
- **（③）** 跨生成器掉点必须在**真实数据**上测；若真实上崩得厉害，正是第二篇泛化论文的动机（第一篇写成「被刻画的局限」）。

### Gate 4 — 评测骨架：全局检测分数对局部编辑迟钝 + 单类/小样本
**现象**：各 split 检测 AUC 波动极大——test_a **0.56**（近随机）、test_b 0.83、test_c 0.75、test_d **0.98**、test_e **无**（单一类别，n=3）、test_f FPR 0.0（n=2）。

**根因（②）**：
- 检测分数 = **整图 residual_map 均值**，对**整图生成/醒目改动**灵敏、对**细微局部重绘**迟钝。故含细微 D2 inpaint 的 `test_a`（fake=whole 5+partial 7）仅 0.56，而 `test_d`（全 partial 但为醒目改动）到 0.98——**检测难度随「改动显著度」变化，与 whole/partial 标签不完全一致**。
- `test_e`（3 张退化 fake，**全为 fake**）、`test_f`（2 张真实，全为 real）→ **组内单一类别 → AUC 无定义**。这是 `gate4_eval` 的**口径问题**：退化/真实-only split 应与**共享真实负样本池**对比，不能组内算。
- 小样本（test_c=4、test_e=3、test_f=2）→ 指标噪声大、不稳。

**优化建议**：
- **（②现在可做）** 检测分数改用**局部最大残差区**或**定位派生分数**（对局部编辑更灵敏），替代全局均值。
- **（②）** `test_e/test_f` 改为与**训练集真实负样本共享池**对比：test_e 报「退化 fake vs 真实」AUC + 相对 test_a 的鲁棒性掉点；test_f 报 FPR（已做）。
- **（②）** 提规模（`probe.yaml` 的 `n_base`→150、`pipeline` 的 `d0` 增大），让每个 split ≥ 数十张，指标才稳。

## 3. 跨闸门共性问题（根因收敛）

| 共性问题 | 波及 | 优化方向 |
|---|---|---|
| **特征太糙**（整图 mean/std） | gate2/3/4 | 加空间/区域残差描述子 + 分位数 + 定位派生分数 |
| **分类器不匹配序数信号**（NearestCentroid） | gate1 | 回归→分桶 / 有序分类器 |
| **小样本 split** | gate4（也影响所有 verdict 稳定性） | 提 `n_base`/`d0` 规模 + bootstrap 置信区间 |
| **mock 信号非真实** | gate1 假阳性 ρ、gate3 无掉点假象、gate2 低估 | 真实 SD2 提取器 + 真实数据（唯一能定论的路径） |

## 4. 优化建议清单（按优先级）

**P0 — checking 分析代码，现在即可做且可测（不需 GPU）**
1. `gate1`：从 `linear_fit_predict` 预测强度反推分桶得 ba，与直接分类取优 → 对齐 ba 与 ρ。
2. `extractor`：`profile` 扩展为 **区域特征**（高残差区重心/半径/边界-中心比 + 残差分位数 p10/p50/p90）。
3. `gate2/gate3`：基于新特征重评；加 img2img-vs-局部 二分层。
4. `gate4_eval`：`test_e/test_f` 用共享真实负样本池；检测分数改局部/定位派生；提规模。
5. 各闸门加 **bootstrap 置信区间**（小样本护栏），VERDICT 附 CI。

**P1 — 出真实科学判定的关键路径（需 GPU/权重，本环境跑不了）**
6. 实现 `checking/extractor.py:DiffusersSD2Residual.residual_stack`（冻结 SD2 的真实多 σ Tweedie 残差）。
7. 接真实扩散后端/真实底图产数据（`backends/real/*` 现为 stub、diffusers 未装）。
8. `--extractor real` 在 GPU 上重跑 gate1–4，才是可写进论文的 go/no-go。

**P2 — 增强**
9. `gate2` 用真正的 t-SNE（现为 PCA-2D）——可选装 sklearn。
10. `report.json` 增加逐 split/逐 family 明细与图表。

## 5. 结论

- **代码层面**：gate1–4 分析在本仓端到端跑通、指标/VERDICT 结构正确（101 tests 全绿）。本次最有价值的产出不是这些 mock VERDICT，而是它们**暴露的分析设计可改进点**：特征太糙（gate2/3/4）、分类器不匹配（gate1）、评测口径与规模（gate4）。上面 **P0** 五条都能立即在 CPU 上落地并复测。
- **科学层面**：本次 VERDICT **不能**作为论文承重假设的证实/证伪。gate1 的 ρ=0.89、gate3 的「无掉点」都带 mock 假象。真正判定必须走 **P1**（真实 SD2 + 真实数据 + GPU）。

## 6. P0 优化后复测（commit 后同数据同代理，仅换分析代码）

已落地 P0：区域/分位特征、gate1 回归→分桶、gate4 共享真实负样本池 + 局部检测分数、bootstrap 置信区间。同一 `data/run`/`data/probe`、同 `multisigma` 代理复测：

| 闸门 | 优化前 | 优化后 | 改进来源 |
|---|---|---|---|
| gate1 | WEAK · ba 0.472（ρ 0.89） | **PASS** · ba **0.757** CI[0.68,0.82]（ρ 0.88） | 回归→分桶：direct 0.49 → reg-bucket **0.757** |
| gate2 | WEAK · same/cross 0.33 | WEAK · same **0.454** / cross **0.478** | 区域特征 (+0.13)，仍未过 0.50 |
| gate3 | PASS · 多σ增量 +0.132 | PASS · 多σ增量 **+0.417** | 富特征使多σ ≫ 单σ |
| gate4 | test_e = **null**（单类） | test_e **0.537** + 各 split 有 AUC 与 **CI** | 共享真实负样本池 + 局部检测分数 |

**解读（区分真实改进 vs mock 限制）**：
- **gate1 WEAK→PASS 是真实的分析改进**：mock 强度与残差单调（ρ 高），改用「回归预测→分桶」后 ba 与 ρ 对齐（0.47→0.76）。但 **ρ 本身仍是 mock 线性混合的产物**，真实 SDEdit 上会退化——PASS 只说明「代码口径对了」，非「t0 逆估计成立」。
- **gate2 +0.13 仍 WEAK**：区域几何特征（重心/边界-中心比）有效但不足以分开 5 算子；需更强空间描述子或真实扩散签名。
- **gate3 增量放大到 +0.42**：富特征下多 σ 相对单 σ 优势更明显（更支持「多尺度」措辞）；跨生成器仍无真实 gap（mock 限制未变）。
- **gate4 口径修复**：`test_e/f` 由「组内单类无定义」改为「与训练真实负样本共享池对比」，`test_e` 从 null → 0.537；各 split AUC 带 bootstrap CI（小样本护栏，如 test_c n_fake=2 → CI 很宽）；局部检测分数使 AUC 更均衡可比。

**诚实边界不变**：以上证明**分析代码更稳健、评测口径更对**；但所有 VERDICT 仍是 **mock + CPU 代理**，非科学结论。写进论文的判定仍须走 **P1**（真实 `DiffusersSD2Residual` + 真实扩散数据 + GPU）。
