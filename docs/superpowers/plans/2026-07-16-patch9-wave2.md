# PATCH 9 Wave 2 实施计划（9.1 nuisance/强度采样 + 9.2 prompt bank/面积分桶/分辨率组 + V11/V12 + gate2 冻结 config）

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把"写进像素后不可回溯"的三类采样政策（nuisance/强度/prompt/掩码面积/分辨率组）落成 config 驱动的 builder 行为 + V11/V12 校验 + gate2 设计冻结 config——B3 阻断的最后一层本地代码。全 mock 可测；SDXL 真实映射只写代码，GPU 冒烟另行。

**Architecture:** 政策全部进 `PipelineConfig`（零硬编码，9.7 验收）；确定性采样一律 `stable_hash(iid+salt)` 派生；nuisance/prompt 记 `op_params`（JSON），`prompt_bank_version`=bank 文件 sha1 前 12 位；D2 补 operator 映射与面积分层；新 `build_grid`（img2img 连续强度 + outpaint 边带网格）供主 run 算子轴；多分辨率组靠 `load_and_resize(size)` 接线 + **V2 集合相等天然承担 9.2c 断言**（real 与 fake 的 `rs{N}>png` 链集合逐 split 必须相等，无需改 V2）。

**Tech Stack:** 纯现有依赖；TDD；mock 后端全链 CPU 测试。
